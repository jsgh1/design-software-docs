# Historias de usuario

> Autor: Equipo del proyecto | Equipo: Analisis / Producto

## Historias principales

| ID | Historia | Criterios de aceptacion |
|---|---|---|
| HU-001 | Como administrador quiero gestionar usuarios y roles para controlar acceso al sistema. | Crear usuario, asignar rol, desactivar sesion, auditar cambios |
| HU-002 | Como coordinador quiero gestionar fichas, programas y RAP para planear formacion. | CRUD academico, validaciones, eventos de cambio |
| HU-003 | Como coordinador quiero conocer disponibilidad de ambientes para evitar asignaciones invalidas. | Consultar disponibilidad, reservar, bloquear por mantenimiento |
| HU-004 | Como coordinador quiero crear horarios sin conflictos para publicar programacion confiable. | Validar instructor, ambiente, ficha y franja |
| HU-005 | Como instructor quiero consultar mis horarios para planificar actividades. | Consulta por instructor y fecha |
| HU-006 | Como aprendiz quiero consultar horarios de mi ficha para asistir a sesiones. | Consulta por ficha y jornada |
| HU-007 | Como administrador documental quiero generar documentos desde plantillas para soportar procesos. | Generar PDF, versionar documento, registrar auditoria |
| HU-008 | Como gestor de calidad quiero ver alertas y KPIs para hacer seguimiento. | Indicadores, alertas, sesiones de seguimiento y plan de mejora |
